Conceptual Framework
Input
  • RGB color spectral readings from Philippine Peso banknotes (TCS34725)
  • RFID UID scans from tagged cards and items (RC522)
  • BLE RSSI signal strength from smartphone
  • Timestamp of each insertion / removal event
  • User account and wallet profile data
  • GPS coordinates via mobile app (last known location)
→
Process
  • On-device K-NN / TFLite AI model classifies banknote denomination from RGB values
  • ESP32 firmware detects RFID tag insert / remove events and weight threshold triggers
  • BLE RSSI computation determines proximity; separation alert logic evaluates threshold
  • Event payloads (denomination, UID, timestamp) packaged and transmitted to Firebase via Wi-Fi
  • ESP32 deep sleep duty-cycle manages power consumption between events
  • Firebase Cloud Functions process incoming events and dispatch FCM push notifications
  • Mobile app aggregates and visualizes data from Firestore real-time listeners
→
Output
  • Identified Philippine Peso denomination per bill inserted (₱20 – ₱1,000)
  • Real-time wallet content list (cards + cash total value)
  • Separation alert push notification when wallet exceeds BLE range
  • Chronological spending history log with denomination and timestamp
  • Cloud-synchronized data accessible across multiple devices
  • Analytics dashboard showing cash flow trends and usage patterns
  • Geospatial last-seen location of the wallet
